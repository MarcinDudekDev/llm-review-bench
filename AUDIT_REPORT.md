# Audit Report ‚Äî `llm-review-bench`

## 1. Summary

This audit covered two surfaces of the public `llm-review-bench` project: (a) a **fact-check** of every quantitative and methodological claim in `blog/article.html` and `README.md` against the raw benchmark data (`results/ALL_RESULTS.json`, `v3_runs.json`, `v4_runs.json`, `oneshot_enforced_summary.json`, `sonnet_haiku_summary.json`), and (b) a **correctness review** of the 9 Python files in `harness/` (`adapters.py`, `run_bench.py`, `run_v3.py`, `run_v4.py`, `exec_score.py`, `consolidate.py`, `chart.py`, `judge.py`, `adversarial_filter.py`). The project did **not** pass clean. After adversarial verification, **26 findings are confirmed**: **4 factual/expository discrepancies** in the writeup and **22 code defects** in the harness (including several duplicates of the same root cause across call sites). Headline result: the harness has **4 high-severity correctness bugs** ‚Äî a dead fork-bomb guardrail that executes model-generated fork bombs, a `grok_cli` adapter that silently scores failed runs as real zero answers, an unguarded `None`-text crash that discards an entire run, and an adversarial-filter extraction bug that mis-promotes tasks into v4. The blog's abstract also overstates two data claims that its own body text correctly qualifies.

## 4. Severity ranking ‚Äî fix these first

**Tier 1 ‚Äî HIGH (fix immediately):**
1. `exec_score.py:20` ‚Äî dead fork-bomb guard; model-generated fork bombs execute and can hang the host. Security/safety guardrail is fully bypassed.
2. `adapters.py:37‚Äì43` `grok_cli` ‚Äî silent failure pollutes contender aggregates *and* corrupts judge verdicts across three call sites; single highest-leverage fix.
3. `run_v3.py:70` (and the parallel `run_v4.py:60` / `adapters.py:61` crash) ‚Äî an unguarded `None` text aborts the run and **discards every collected trial**; data-loss on a condition that occurs in normal provider operation.
4. `adversarial_filter.py:42‚Äì45` ‚Äî last-line extraction silently mis-promotes tasks into v4, corrupting the benchmark's survivor set (the core v4 output).

**Tier 2 ‚Äî MEDIUM (fix before publishing corrected numbers):**
5. `consolidate.py:44` ‚Äî infra failures deflate exec accuracy (directly affects headline scores).
6. `adversarial_filter.py:39‚Äì40/88` ‚Äî solver infra failures inflate v4 survivors (same corpus-integrity concern).
7. `adapters.py:53‚Äì61` (OpenRouter cost never requested) + `run_bench.py:38` (OpenRouter timeouts misclassified) ‚Äî both silently corrupt OpenRouter reporting.
8. `exec_score.py:37` ‚Äî exit code unchecked; false full-pass scores.
9. `run_v4.py:28` backtick strip; `judge.py:67‚Äì68` LETTERS cap; `chart.py:37` missing-metric-as-zero; `exec_score.py` `\nc` over-match ‚Äî real defects, several latent on current config/data.
10. **Blog/README factual (medium):** align the abstract's "the slowest" and "cheaper per run" claims with the body/data, and add the v4 methodology reconciliation. These are visible, verifiable overstatements in a public writeup.

**Tier 3 ‚Äî LOW (cleanup):** the two cost `n/a` truthiness bugs, `--models` empty/`KeyError`/whitespace handling, whitespace-only command, `-nc` flag over-match, `chart.py:41` zero-label, and the README `143`‚Üí`142` rounding fix.
